Production-style Playwright automation framework for validating the critical user journeys of [SauceDemo](https://www.saucedemo.com/), a sample e-commerce application used to demonstrate robust QA automation practices.

## Project Overview

This framework solves the need for fast, repeatable regression feedback across the most business-critical areas of an e-commerce workflow: authentication, product discovery, cart management, checkout, and UI consistency.

Playwright was selected because it provides reliable browser automation, first-class TypeScript support, cross-browser execution, isolated browser contexts, built-in reporting, tracing, screenshots, and video diagnostics. These capabilities make it well suited for scalable QA automation in modern CI/CD environments.

The main objectives of the automation framework are:

- Detect regression risks early through automated end-to-end and UI checks.
- Improve release confidence by covering high-value user journeys.
- Keep test code maintainable through Page Object Model and reusable components.
- Support cross-browser and responsive validation for broader quality coverage.
- Provide actionable failure diagnostics through reports, traces, screenshots, and videos.
- Run consistently in local development and GitHub Actions CI/CD.

## Features

- End-to-End UI automation for complete business workflows.
- Page Object Model (POM) for maintainable page-level abstractions.
- Cross-browser testing across Chromium, Firefox, WebKit, and mobile device profiles.
- Parallel execution using Playwright's project and worker model.
- GitHub Actions CI/CD for automated regression execution.
- HTML reporting for clear test run analysis.
- Screenshots on failure for faster visual debugging.
- Trace and video collection for step-by-step failure investigation.
- Reusable test utilities through fixtures, page objects, component objects, and JSON test data.
- Visual snapshot coverage for layout and responsive regression checks.
- Auth state setup to reduce duplicate login steps across dependent tests.

## Framework Architecture

The framework is organized around maintainability, clear separation of concerns, and scalable test coverage.

- `tests/` contains executable Playwright test suites grouped by testing objective.
- `tests/pages/` contains Page Object Model classes for page-level behavior and assertions.
- `tests/components/` contains reusable component abstractions for shared UI elements.
- `tests/fixtures/` contains typed models, authentication setup, device dimensions, and reusable data dependencies.
- `tests/fixtures/data/` acts as the test-data layer for users, products, carts, and checkout inputs.
- `utils/` is the recommended location for future generic helpers as framework complexity grows.
- `playwright-report/` is generated after execution and contains the HTML report.
- `test-results/` is generated by Playwright and stores failure artifacts such as traces, screenshots, and videos.
- `.github/workflows/` contains CI/CD workflows for regression execution and visual snapshot maintenance.

## CI/CD Pipeline

The GitHub Actions workflow runs automated Playwright tests as part of the development lifecycle.

- Automated execution on push to `main`.
- Automated execution on pull request targeting `main`.
- Node.js setup and dependency installation with `npm ci`.
- Browser installation with `npx playwright install --with-deps`.
- Test execution with `npx playwright test`.
- HTML report generation through Playwright's reporter.
- Artifact storage for the `playwright-report/` directory.
- GitHub Pages deployment for published report visibility.
- Dedicated snapshot update workflow for controlled visual baseline maintenance.

## Reporting

Playwright provides several layers of reporting and diagnostics:

- HTML Report: interactive report with test results, duration, project metadata, and failure details.
- Trace Viewer: full execution trace for reviewing actions, selectors, network activity, screenshots, and DOM snapshots.
- Screenshots: captured automatically on test failure to support fast visual triage.
- Videos: retained for failed tests to replay the browser session and understand the failure sequence.
- CI Artifacts: uploaded by GitHub Actions so results can be reviewed after remote execution.

## How To Run

Install dependencies:

```bash
npm install
```

Install Playwright browsers:

```bash
npx playwright install
```

Run the full test suite:

```bash
npx playwright test
```

Open the HTML report:

```bash
npx playwright show-report
```

Optional targeted runs:

```bash
npx playwright test tests/e2e
npx playwright test tests/ui
npx playwright test tests/visual
```
